# Auth included in the boilerplate as the first vertical slice

The boilerplate ships with working email and password authentication via Supabase Auth as its first real vertical slice. This was the natural choice: auth is the first thing every project built on this template will need, and implementing it correctly demonstrates every key architectural pattern in one coherent, useful feature — boundary parsing (sign-in/sign-up form input), Server Actions (form submission, session handling), Drizzle + Supabase auth schema compatibility, RLS policies protecting user data, and the Container/Presentation split in a real UI. OAuth providers and magic links are out of scope for the starter — they can be added per-project. A `.env.example` is committed to the repository listing every required environment variable with a description, so no developer cloning the template is left guessing what needs to be configured.
