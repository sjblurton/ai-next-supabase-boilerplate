# Storybook for component development with Playwright visual regression in CI only

Storybook is included for developing and verifying Presentation Components, Atoms, Molecules, and Organisms in isolation. Each Story covers one distinct component state and includes a `play` function for interaction testing — this replaces standalone unit tests for pure UI behaviour, since the Story is both the documentation and the test. Visual regression runs via Playwright in CI only, not locally. Committed baseline screenshots are stored in the repository; the CI job compares the current build against them and flags any visual diff for human review. Running visual regression locally was rejected because it requires baseline management per developer machine, is slow, and generates noise. CI-only baselines are authoritative — if a visual change is intentional, the developer updates the baseline screenshots as part of the PR. Accessibility checks are included in the Storybook play functions using the `@storybook/addon-a11y` integration.
