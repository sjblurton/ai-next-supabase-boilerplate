# Prettier for formatting and import organisation over ESLint import sorting

ESLint import-sorting rules and Prettier conflict by default — enabling both results in either Prettier overriding ESLint's order silently, or ESLint producing warnings that cannot be auto-fixed without breaking Prettier's output. The decision was made to delegate all formatting, including import organisation, to Prettier via `prettier-plugin-organize-imports`. ESLint handles code correctness only; Prettier handles all style and ordering. This removes the conflict entirely and keeps the two tools in their natural roles. `prettier-plugin-organize-imports` uses the TypeScript language service to sort imports, which is compatible with the project's strict TypeScript setup. Husky + lint-staged runs Prettier on staged files at pre-commit so formatting issues are caught locally before CI.
