# CI quality gates, branch protection, and pre-commit hooks

Main branch is protected — all changes must arrive via PR. Direct pushes are blocked. A GitHub Actions CI pipeline runs on every PR in the following fixed order: `tsc --noEmit` → ESLint → Vitest with coverage → Playwright visual regression. The order is deliberate: type errors are cheapest to catch first; coverage and visual regression are most expensive and only run when earlier gates pass. Test coverage must never drop below 80% (line and branch) — this is the hard floor. The floor is treated as a ratchet: once the codebase exceeds it, the threshold in CI is raised to match, so coverage can only ever increase over the project lifetime. Pre-commit hooks run via Husky + lint-staged: staged files are linted and formatted by Prettier before the commit lands. A pre-push hook runs `tsc --noEmit` to catch type errors before the branch reaches CI. This catches the majority of issues locally, keeping CI fast by not surfacing easily-avoided failures in the pipeline.
